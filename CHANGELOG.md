# Changelog

## 0.1.0

- Legacy NVIDIA OpenCL: after CLBlast + `set_rows` + `legacy_core.cl`, init runs the full `load_cl_kernels` pass at OpenCL C 1.2. Failed program builds log and skip (null program/kernel) instead of aborting, so Kepler can pick up any kernels the driver compiles. `supports_op` and `ggml_cl_compute_forward` delegate to the standard OpenCL op matrix for everything except CLBlast-routed `MUL_MAT` (still gated by `ggml_opencl_can_mul_mat_legacy`). `ARGSORT` support checks require a non-null argsort kernel after partial loads.
- Legacy NVIDIA OpenCL: `GET_ROWS` for token embeddings on GPU (`F32`, `Q4_0`, `Q6_K` indices `I32`, contiguous; Q4/Q6 require `ne[0]` divisible by 32 / 256) via new `legacy_core.cl` kernels without `cl_khr_fp16`. `MUL_MAT` tries CLBlast first, then **native OpenCL 1.2 FP32 GEMM** on the same device if CLBlast fails (e.g. Kepler `clBuildProgram` / driver quirks) instead of aborting; `GGML_OPENCL_LEGACY_NATIVE_GEMM=1` skips CLBlast entirely. Init calls `clblast::FillCache(device)` to improve first-kernel success on tuned databases.
- Fixed Kepler fork CI (Linux/Windows): `ggml_cl_legacy_*` helpers that use `ggml_tensor_extra_cl` are defined only after that struct in `ggml-opencl.cpp`, restoring CLBlast+legacy builds that failed with undeclared identifier / MSVC syntax errors near the top of the translation unit.
- Kepler fork OpenCL legacy path: added embedded `legacy_core.cl` (OpenCL 1.2, FP32, no subgroups) for `RMS_NORM`, `ROPE` (normal/NeoX; not MROPE/vision/IMROPE), `SOFT_MAX` (FP32 mask), `DIAG_MASK_INF`, `ADD`/`MUL`/`SCALE`, `CPY`/`DUP`/`CONT` (F32 contiguous), and unary `SILU`/`GELU` (tanh); wired `supports_op` / `ggml_cl_compute_forward`. Device description appends `[fork_kepler_opencl]` and `[opencl_legacy_no_fp16]` when applicable. `llama_context` promotes F16 K/V to F32 when FP16 is unavailable, disables Flash Attention on this path, and `llama_kv_cache` no longer forces KV to CPU for tagged devices. Set `GGML_OPENCL_LEGACY_OP_INVENTORY=1` to log unsupported ops during scheduling probes.
- OpenCL backend: stop calling `clReleaseContext` on last backend free by default. The shared context from device probe is reused via cached `ggml_cl2_init` state; releasing it during model load/teardown caused `CL_INVALID_CONTEXT` (-34) on legacy NVIDIA (e.g. driver 411.x). Opt-in with `GGML_OPENCL_RELEASE_CONTEXT=1` for explicit release; document lifecycle in `docs/backend/OPENCL.md` and `docs/kepler-opencl-fork.md`.
- Fixed Kepler fork CI: `ggml_opencl_legacy_col_chunking_feasible` now takes `const ggml_backend_opencl_context *` so `ggml_opencl_can_mul_mat_legacy` compiles under MSVC and GCC (`-fpermissive` / C2664).
- Legacy NVIDIA / CLBlast `MUL_MAT` path: dequantize weights in column slices when a single FP32 staging buffer would exceed `CL_DEVICE_MAX_MEM_ALLOC_SIZE` (fixes OpenCL `-4` / allocation failure on 1 GiB max-alloc GPUs with large matrices such as big vocab embeddings); `supports_op` rejects shapes that cannot be tiled under device alignment and dequant work-group rules.
- Added a legacy NVIDIA / Kepler OpenCL compatibility profile that accepts OpenCL 1.2 devices, routes validated `MUL_MAT` workloads through CLBlast, and falls back to CPU for unsupported operations.
- Replaced the upstream multi-backend build matrix with a fork-specific GitHub Actions workflow and CMake presets focused on the Kepler OpenCL / CLBlast path, and documented the fork maintenance process under `docs/`.
- Fixed the fork-specific Kepler OpenCL CI by restoring the missing Windows OpenCL SDK setup and resolving the C++ forward-declaration break in the legacy CLBlast path.
- Fixed additional Kepler OpenCL CI regressions by guarding OpenCL 2.x/3.0-only profiling and buffer-allocation APIs when building the forced OpenCL 1.2 compatibility target.
- Added automated GitHub Release publishing for successful `master` builds when `CHANGELOG.md` contains a real version heading, using auto-generated `build-<run_number>` tags and attaching both platform artifacts.
- Fixed the legacy Kepler runtime scheduler mismatch by enabling OpenCL `SET_ROWS` support for KV-cache writes in the CLBlast compatibility path.
- Fixed legacy NVIDIA runtime startup on OpenCL 1.2 drivers without FP16 support by skipping FP16 `SET_ROWS` kernel compilation and keeping incompatible F16 KV-cache buffers on CPU.
- Fixed OpenCL device `ggml_backend_dev_description` to include the OpenCL C version (e.g. `name (OpenCL 1.2 CUDA)`), so the KV-cache CPU fallback that detects legacy OpenCL 1.2 actually triggers instead of leaving F16 caches on GPU and aborting on `SET_ROWS`.
- Replaced the old upstream-oriented `AGENTS.md` with fork-specific persistent guidance covering this fork's Kepler/OpenCL goal, key files, maintenance workflow, and mandatory changelog/docs updates after changes.
